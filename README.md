# Virtual Queue API — Sprint 1 (Foundation)

## Why you need to run a few commands locally

This solution was authored in an environment with no internet access to
NuGet or the .NET SDK, so package restore, build, and EF migration
generation could not be executed here. Every file was hand-written and
reviewed for consistency, but **you must run the commands below on your
own machine** (with .NET 8 SDK installed) to restore, build, generate the
first migration, and run it.

## Prerequisites

- .NET 8 SDK
- One of:
  - SQL Server LocalDB (ships with Visual Studio on Windows), or
  - SQL Server in Docker (Mac/Linux):
    ```bash
    docker run -e "ACCEPT_EULA=Y" -e "MSSQL_SA_PASSWORD=YourStrong!Passw0rd" \
      -p 1433:1433 --name sql1 -d mcr.microsoft.com/mssql/server:2022-latest
    ```
    If you use Docker, update `appsettings.json`'s connection string to:
    `Server=localhost,1433;Database=VirtualQueueDb;User Id=sa;Password=YourStrong!Passw0rd;TrustServerCertificate=True`
- `dotnet-ef` CLI tool: `dotnet tool install --global dotnet-ef`

## 1. Restore and build

```bash
cd VirtualQueue
dotnet restore
dotnet build
```

## 2. Generate the initial migration

This was intentionally **not** pre-generated by hand — hand-writing an EF
Core migration risks drifting from what the real EF Core scaffolder
produces (column ordering, default constraint names, etc.). Generate it
for real:

```bash
cd src/VirtualQueue.Infrastructure
dotnet ef migrations add InitialCreate --startup-project ../VirtualQueue.Api
```

## 3. Apply the migration (creates the database)

```bash
dotnet ef database update --startup-project ../VirtualQueue.Api
```

## 4. Run the API

```bash
cd ../VirtualQueue.Api
dotnet run
```

Browse to `https://localhost:5081/swagger` (or the URL printed in the
console) to see the Swagger UI. Try:
- `GET /api/health` → confirms the API is running
- `GET /api/health/db` → confirms the database connection works

## 5. Run the tests

```bash
cd ../../tests/VirtualQueue.Tests
dotnet test
```

You should see all Domain tests (Business, Service, QueueEntry state
machine) pass.

## Project structure

```
VirtualQueue.sln
├── src/
│   ├── VirtualQueue.Domain          → entities, enums, domain exceptions (no dependencies)
│   ├── VirtualQueue.Application     → interfaces (IApplicationDbContext), DTOs with validation
│   ├── VirtualQueue.Infrastructure  → EF Core DbContext, Fluent API configurations, migrations
│   └── VirtualQueue.Api             → controllers, middleware, DI composition root, Swagger
└── tests/
    └── VirtualQueue.Tests           → xUnit + FluentAssertions unit tests
```
